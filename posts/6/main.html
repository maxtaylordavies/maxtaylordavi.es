1. The problem

As humans navigating the world, we frequently find ourselves acting within environments populated by various different agents (for us, typically other humans). In the course of this, we often want (for whatever reason) to construct some mental model of these other agents by observing their behaviour. To do this, we could treat the agent population as simply a collection of individuals, considering each one independently in turn. But, as we know from our own experience, populations of agents usually contain some amount of social structure; furthermore, this social structure often encodes information that is useful to us as an observer. So as we engage in our agent modelling, we probably want to do so in a way that accounts for any social structure present, rather than ignoring it.

To formalise the problem a bit, let's say that we're in an environment populated by MMM other agents. We have access to an observation history O\mathbf{O}O, where OmO_mOm​ is some representation of agent mmm's behaviour up to the current timestep. We think that the MMM agents can be partitioned into an unknown number KKK of latent groups, based solely on their behaviour. Our task is to infer (from O\mathbf{O}O) a partition z\mathbf{z}z that assigns each agent to a latent group.

2. A general approach

In general, latent social structure models have the following form, based on more general Bayesian approaches to structure learning \citep{Tenenbaum2011HowTG, Austerweil2015StructureAF}. Let O\mathbf{O}O be a set of observations, and let z\mathbf{z}z be a partition of agents into groups. Then, by Bayes' rule,

P(z∣O)∝P(O∣z)P(z)\begin{equation} P(\mathbf{z}|\mathbf{O}) \propto P(\mathbf{O}|\mathbf{z})P(\mathbf{z}) \end{equation}P(z∣O)∝P(O∣z)P(z)​​

with the likelihood P(O∣z)P(\mathbf{O}|\mathbf{z})P(O∣z) given by marginalising over the latent parameters θ\mathbf{\theta}θ:

P(O∣z)=∫θP(O∣θ,z)P(θ)dθ\begin{equation} P(\mathbf{O}|\mathbf{z}) = \int_\theta P(\mathbf{O}|\mathbf{\theta},\mathbf{z})P(\mathbf{\theta})d\mathbf{\theta} \end{equation}P(O∣z)=∫θ​P(O∣θ,z)P(θ)dθ​​

where P(O∣θ,z)P(\mathbf{O}|\mathbf{\theta},\mathbf{z})P(O∣θ,z) and P(θ)P(\mathbf{\theta})P(θ) will depend on the particular generative model that is specified.

3. Example: binary choice data

Let's illustrate the approach here by applying it to a very simple form of the problem - inferring group assignments from observations of discrete choices. To begin with we will restrict all the choices to be binary; we will later relax this to consider choices over arbitrary numbers of options. Consider a population of MMM agents making a series of NNN independent binary choices between options AAA and BBB. This tells us how to compute the probability of the observed choice history given some candidate partition z\mathbf{z}z. But to complete equation 1, we also need to define a prior over partitions P(z)P(\mathbf{z})P(z). How should we do this? A sensible choice is the Chinese Restaurant Process (CRP) prior \citep{Blei2003HierarchicalTM}, which permits a theoretically unbounded number of groups to be specified while favouring partitions that invoke fewer groups.

P(z∣c)=cKΓ(c)∏kΓ(Tk)Γ(M+c)\begin{equation} P(\mathbf{z}|c) = \frac{c^K\Gamma(c)\prod_k\Gamma(T_k)}{\Gamma(M + c)} \end{equation}P(z∣c)=Γ(M+c)cKΓ(c)∏k​Γ(Tk​)​​​

where c≥0c \geq 0c≥0 is the concentration parameter (which controls the tendency of the CRP to create new groups), and TkT_kTk​ is (as before) the number of agents assigned to group kkk. The advantage of the CRP is that, by allowing the number of groups to be determined by the observations provided, it grants the model a level of flexibility not present in those that assume a fixed number of groups or categories. This is important for social structure learning, where the number of groups present within an environment or population can be highly variable - for instance, in some situations there may only be a single group containing all agents; in others, every agent may represent a different unique group.

Given a set of candidate partitions, we can compute a posterior distribution over them following equation 6:

python
def posterior(O, partitions, a, b, c):
    likelihoods = np.array([likelihood(O, z, a, b) for z in partitions])
    z_priors = np.array([crp(z, c) for z in partitions])
    probabilities = np.multiply(likelihoods, z_priors)
    return probabilities / np.sum(probabilities)

where the likelihood and prior functions are written following equations 4 and 5 (using the betabinomial implementation from scipy.stats):

python
def likelihood(O, z, a, b):
    totals = Counter(z)
    return np.prod(
        [np.prod(betabinom.pmf(np.sum(O[z == k], axis=0), totals[k], a, b)) for k in totals]
    )

def crp(z, c):
    counts = Counter(z)
    prod = np.prod([gamma(counts[k]) for k in counts])
    coeff = ((c ** len(counts)) * gamma(c)) / gamma(c + len(z))
    return coeff * prod

To test our model, we would ideally like to have it compute a distribution over all possible partitions of the agents. But since our model does not assume a fixed KKK, the total number of possible partitions is given by MM=1010M^M = 10^{10}MM=1010, which is clearly prohibitively large. For practicality's sake, we will limit ourselves to considering only the set of possible partitions with K≤3K \leq 3K≤3. Since our CRP prior will push down the probability of partitions with large KKK, this should actually have little effect on the resulting distribution.

python
def generate_all_partitions(M):
    partitions = set()
    for i in range(0, M + 1):
        k = M - i
        for j in range(0, k + 1):
            base = ([0] * i) + ([1] * j) + ([2] * (k - j))
            partitions = partitions.union(set(permutations(base)))
    return np.array(partitions)

4. Arbitrary choice size and reward functions

Having developed a model for binary choice observations, we now consider how it can be extended to account for unbounded choice sizes. At each choice trial, agents now choose between LLL different options. Rather than being a scalar, θk\theta_kθk​ becomes a vector θk=[θk1,...,θkL]\mathbf{\theta}_k = [\theta_{k1}, ..., \theta_{kL}]θk​=[θk1​,...,θkL​] parameterising a multinomial distribution. In place of our beta prior on group parameters, we turn to its multivariate generalisation, the Dirichlet distribution.
