1. The problem

As humans navigating the world, we frequently find ourselves acting within environments populated by various different agents (for us, typically other humans). In the course of this, we often want (for whatever reason) to construct some mental model of these other agents by observing their behaviour. To do this, we could treat the agent population as simply a collection of individuals, considering each one independently in turn. But, as we know from our own experience, populations of agents usually contain some amount of social structure; furthermore, this social structure often encodes information that is useful to us as an observer. So as we engage in our agent modelling, we probably want to do so in a way that accounts for any social structure present, rather than ignoring it.

To formalise the problem a bit, let's say that we're in an environment populated by MMM other agents. We have access to an observation history O\mathbf{O}O, where OmO_mOm​ is some representation of agent mmm's behaviour up to the current timestep. We think that the MMM agents can be partitioned into an unknown number KKK of latent groups, based solely on their behaviour (where each group is characterised by a parameter value θk\theta_kθk​). Our task is to infer (from O\mathbf{O}O) a partition z\mathbf{z}z that assigns each agent to a latent group (i.e. a vector of length MMM where zmz_mzm​ gives the group assignment of agent mmm).

2. A general approach

In general, latent social structure models have the following form, based on more general Bayesian approaches to structure learning \citep{Tenenbaum2011HowTG, Austerweil2015StructureAF}. Let z\mathbf{z}z be some candidate partition of agents into groups. Then, by Bayes' rule,

P(z∣O)∝P(O∣z)P(z)\begin{equation} P(\mathbf{z}|\mathbf{O}) \propto P(\mathbf{O}|\mathbf{z})P(\mathbf{z}) \end{equation}P(z∣O)∝P(O∣z)P(z)​​

with the likelihood P(O∣z)P(\mathbf{O}|\mathbf{z})P(O∣z) given by marginalising over the latent parameters θ\mathbf{\theta}θ:

P(O∣z)=∫θP(O∣θ,z)P(θ)dθ\begin{equation} P(\mathbf{O}|\mathbf{z}) = \int_\theta P(\mathbf{O}|\mathbf{\theta},\mathbf{z})P(\mathbf{\theta})d\mathbf{\theta} \end{equation}P(O∣z)=∫θ​P(O∣θ,z)P(θ)dθ​​

where P(O∣θ,z)P(\mathbf{O}|\mathbf{\theta},\mathbf{z})P(O∣θ,z) and P(θ)P(\mathbf{\theta})P(θ) will depend on the particular generative model that is specified.

3. Example: binary choice data

Let's illustrate the approach here by applying it to a very simple form of the problem - inferring group assignments from observations of discrete choices. To begin with we will restrict all the choices to be binary; we will later relax this to consider choices over arbitrary numbers of options. Consider a population of MMM agents making a series of NNN independent binary choices between options AAA and BBB. The observation set O\mathbf{O}O is then a binary matrix such that OmnO_{mn}Omn​ indicates the choice made by agent mmm on trial nnn. Given a candidate partition z\mathbf{z}z, we will use TkT_kTk​ to denote the total number of agents assigned to group kkk and Akn(1)A_{kn}^{(1)}Akn(1)​ for the number of group-kkk agents that chose option 1 on trial nnn. We can then write

This tells us how to compute the probability of the observed choice history given some candidate partition z\mathbf{z}z. But to complete equation 1, we also need to define a prior over partitions P(z)P(\mathbf{z})P(z). How should we do this? A sensible choice is the Chinese Restaurant Process (CRP) prior \citep{Blei2003HierarchicalTM}, which permits a theoretically unbounded number of groups to be specified while favouring partitions that invoke fewer groups.

P(z∣c)=cKΓ(c)∏kΓ(Tk)Γ(M+c)\begin{equation} P(\mathbf{z}|c) = \frac{c^K\Gamma(c)\prod_k\Gamma(T_k)}{\Gamma(M + c)} \end{equation}P(z∣c)=Γ(M+c)cKΓ(c)∏k​Γ(Tk​)​​​

where c≥0c \geq 0c≥0 is the concentration parameter (which controls the tendency of the CRP to create new groups), and TkT_kTk​ is (as before) the number of agents assigned to group kkk. The advantage of the CRP is that, by allowing the number of groups to be determined by the observations provided, it grants the model a level of flexibility not present in those that assume a fixed number of groups or categories. This is important for social structure learning, where the number of groups present within an environment or population can be highly variable - for instance, in some situations there may only be a single group containing all agents; in others, every agent may represent its own unique group.

Given a set of candidate partitions, we can compute a posterior distribution over them following equation 6:

python
def posterior(O, partitions, a, b, c):
    likelihoods = np.array([likelihood(O, z, a, b) for z in partitions])
    z_priors = np.array([crp(z, c) for z in partitions])
    probabilities = np.multiply(likelihoods, z_priors)
    return probabilities / np.sum(probabilities)

where the likelihood and prior functions are written following equations 4 and 5 (using the betabinomial implementation from scipy.stats):

python
def likelihood(O, z, a, b):
    totals = Counter(z)
    return np.prod(
        [np.prod(betabinom.pmf(np.sum(O[z == k], axis=0), totals[k], a, b)) for k in totals]

def crp(z, c):
    counts = Counter(z)
    prod = np.prod([gamma(counts[k]) for k in counts])
    coeff = ((c ** len(counts)) * gamma(c)) / gamma(c + len(z))
    return coeff * prod

python
def generate_all_partitions(M):
    partitions = set()
    for i in range(0, M + 1):
        k = M - i
        for j in range(0, k + 1):
            base = ([0] * i) + ([1] * j) + ([2] * (k - j))
            partitions = partitions.union(set(permutations(base)))
    return np.array(partitions)

4. Simple extensions

4.1 Arbitrary choice size

Having developed a model for binary choice observations, we now consider how it can be extended to account for unbounded choice sizes. At each choice trial, agents now choose between LLL different options. Rather than being a scalar, θk\theta_kθk​ becomes a vector θk=[θk1,...,θkL]\mathbf{\theta}_k = [\theta_{k1}, ..., \theta_{kL}]θk​=[θk1​,...,θkL​] parameterising a multinomial distribution. In place of our beta prior on group parameters, we turn to its multivariate generalisation, the Dirichlet distribution. Since the Dirichlet distribution is conjugate to the multinomial, we can follow the same procedure as before and obtain the likelihood as

P(O∣z)=∏k∏nΓ(Lγ)Γ(Tk+1)Γ(Tk+Lγ)∏lΓ(Akn(l)+γ)Γ(γ)Γ(Akn(l)+1)=∏k∏nDirMultinom(Akn∣Tk,γ)\begin{equation} \begin{split} P(\mathbf{O}|\mathbf{z}) &= \prod_k\prod_n\frac{\Gamma(L\gamma)\Gamma(T_k + 1)}{\Gamma(T_k + L\gamma)}\prod_l\frac{\Gamma(A_{kn}^{(l)} + \gamma)}{\Gamma(\gamma)\Gamma(A_{kn}^{(l)} + 1)} \\ &= \prod_k\prod_n\text{DirMultinom}(\mathbf{A}_{kn} | T_k, \gamma) \end{split} \end{equation}P(O∣z)​=k∏​n∏​Γ(Tk​+Lγ)Γ(Lγ)Γ(Tk​+1)​l∏​Γ(γ)Γ(Akn(l)​+1)Γ(Akn(l)​+γ)​=k∏​n∏​DirMultinom(Akn​∣Tk​,γ)​​​

where Akn=[Akn(1),...,Akn(L)]\mathbf{A}_{kn} = [A_{kn}^{(1)}, ..., A_{kn}^{(L)}]Akn​=[Akn(1)​,...,Akn(L)​] is a vector giving the number of agents assigned to group kkk that chose each of the LLL options on choice nnn, and γ\gammaγ is the parameter of the symmetrical Dirichlet. Using the same CRP prior for partitions as before, our full model is now written as

P(z∣O;γ,c)∝cKΓ(c)∏kΓ(Tk)Γ(M+c)∏k∏nDirMultinom(Akn∣Tk,γ)\begin{equation} P(\mathbf{z}|\mathbf{O}; \gamma, c) \propto \frac{c^K\Gamma(c)\prod_k\Gamma(T_k)}{\Gamma(M + c)}\prod_k\prod_n\text{DirMultinom}(\mathbf{A_{kn}} | T_k, \gamma) \end{equation}P(z∣O;γ,c)∝Γ(M+c)cKΓ(c)∏k​Γ(Tk​)​k∏​n∏​DirMultinom(Akn​∣Tk​,γ)​​

To evaluate this model, we can go through the same procedure as before, changing only the code for generating observations and computing likelihoods.

Because scipy.stats doesn't provide a Dirichlet-Multinomial implementation, we write our own following Equation 8:

python
def dirichlet_multinomial(counts, T, L, g, log=True):
    coeff = (gamma(L * g) * gamma(T + 1)) / gamma(T + (L * g))
    prods = np.prod(gamma(counts + g) / (gamma(g) * gamma(counts + 1)), axis=1)
    tmp = coeff * prods
    return np.sum(np.log(tmp)) if log else np.prod(tmp)

I.e. ϵ\epsilonϵ gives the probability that an agent chooses an option other than the one favoured by their group. Rather than hand-specifying a single true partition like we did before, we'll run model evaluation over a number of repeats, where for each repeat we first randomly sample a partition (from a uniform distribution) and then generate observations according to the sampled partition.

4.2 Reward functions

We will now introduce a slight reframing of this model in a way that will make it more generally applicable later. Rather than choosing between options according to a categorical distribution parameterised by their group, we can think of agents' choices as being determined by a reward function. Strictly speaking, a reward function is a mapping from possible states of the environment to scalar reward values; in our simplified setting we can simply express it as a vector of discrete values such that rl(m)r^{(m)}_lrl(m)​ gived the amount of reward that agent mmm recieves as a result of choosing option lll. We then say that each agent mmm acts (i.e. makes choices) in order to maximise their reward under approximate rationality:

π(m)(l)∝exp⁡(rl(m)β)\begin{equation}\pi^{(m)}(l) \propto \exp(\frac{r^{(m)}_l}{\beta})\end{equation}π(m)(l)∝exp(βrl(m)​​)​​

where the Boltzmann parameter β\betaβ governs the amount of decision noise present; i.e. the probability that the agent selects a non-reward-maximising option. As β→0\beta \to 0β→0, the agent becomes perfectly rational.

For now, we will say that an agent's reward function is entirely determined by their group assignment; i.e. each group is associated with a different reward function, and all members of that group share that same reward function. 5. Gridworlds

So far so good. However, it would be nice if we could extend the discrete-choice setting to be a little more realistic. Instead of having agents make abstract choices, we will now consider them as acting within a simple 2D gridworld environment. At any point within the environment, an agent selects one of four possible actions: {Up, Down, Left, Right}. The effects of actions are deterministic; e.g. as long as there is no obstacle in the way, selecting Down will always move the agent to the square immediately below their current location. The environment contains a number of different objects, corresponding to the choice options in our previous framing. An agent recieves reward by navigating to an object; how much reward each object provides is governed by the agent's reward function r(m)r^{(m)}r(m).

5.1 Operating on choice proxies

Our observation history O\mathbf{O}O will now consist of sets of trajectory observations, represented as (variable-length) sequences of discrete actions, where OmnO_{mn}Omn​ is the nnnth trajectory observed for agent mmm. How should we deal with these observations? One simple approach is to try and extract summary choice variables from the raw observation history, to get an M×NM \times NM×N matrix O^\mathbf{\hat{O}}O^ of estimated choices that we can use in the same way as we used our actual choice matrix in the previous setting. To do this, we can follow a simple heuristic: as long as different choice objects are sufficiently separated in 2D space, then an agent's average direction of travel (from a common starting location) should be a good proxy for which object they chose.
